函数相关的及周边问题总结梳理

​ 1- 函数的作用

​ + 基本定义:把一段功能性的代码封装到函数中,以实现复用的效果,在要使用的时候,直接调用这个函数就好了

​ + 后期还会接触函数吗?

​ - 现在急于js封装的库,或者是插件,都是用函数封装的代码

​ + 工作中:

​ - 团队需要结合实际业务需求,封装一个公用的库,或者是插件的时候,会用到函数封装

​ 2- 创建函数的方式

​ + 函数声明

​ - function fn () {} fn();

​ - 后边我们会经常使用这种方式创建函数

​ + 函数表达式

​ - var fn = function () {};

​ - 看一个具体的使用场景

​ + 匿名函数

​ - (function () {})();

​ - 只能执行一次

​ 3- 函数的参数

​ + 形参

​ - 形参就是形式上的参数,作用就是用来接收实参传递给具体的数值

​ - function fn (形参的位置) {} fn()

​ - 注意的问题:

​ - 当实参没有传递给形参具体数值的时候,就会报出undefined

​ - 实参和形参的个数需要保持一致,当形参多于实参的时候,就会产生undefined

​ + 实参

​ - 实参就是在函数调用的时候,传递给形参的一个具体数值,可以是字符串,数组….

​ - function fn () {} fn(实参的位置);

​ + 函数中的形参是属于全局下的还是局部的??

​ - 形参是属于函数内部声明的变量,故在函数外部无法访问

​ 4- 函数的返回值以及函数的状态

​ + 函数状态:

​ - 有返回值的函数

​ - 无返回值的函数

​ + 函数中声明了变量,接收了形参传入的值,如果没有return,就会报出undefined

​ + 如果使用了return,但是没有明确的返回值,也是undefined

​ + 如果使用return,后续的代码将不再执行’

​ + 函数的几种状态(了解):

​ - 有参数无返回值的函数

​ - 有返回值无参数的函数

​ - 无参数无返回值的函数

​ - 有参数有返回值的函数

​ + return后面还可以去写一个函数

​ - 这样的函数,称之为闭包函数

​ - js高级阶段会讲到闭包函数,是一个重点(较为抽象)

​ - 补充:我已经说了,到时候你看着办吧~~;

​ 5- 作用域以及作用域链

​ + 全局作用域

​ - 在函数外部声明的变量都是属于全局下的变量

​ + 局部作用域

​ - 在函数内部声明的变量就是属于局部变量

​ + 注意事项:

​ - 在函数内部可以访问全局下的变量

​ - 但是在全局下是无法访问到函数内部声明的变量

​ + 小小的拓展:

​ - 在全局下声明了一个变量就是在全局最大的window对象上面,新添加了一属性

​ + 作用域链

​ - 在函数中定义了一个变量,如果当前作用域下存在,那么就输出,否则会沿着当前作用域依次向上查找,这个查找的过程形成了一个链式结构,我们把这个链式结构,叫做’作用域链’

​ - 注意:如果在全局下还没有找到,那么就直接报错

​ 6- js的预解析机制

**** + 预解析定义:

​ - 就是js解析过程中的一种机制,会把变量和函数声明进行提升

​ - 注意:函数声明高于变量声明

​ + 注意事项:

​ - 切记不要重复使用一个函数名字,否则后者会把前者重新赋值

​ - 变量名和函数名字不要冲突

​ - 实际开发中:

​ - 一个单位/项目中会提前规范好命名方式,例如:变量名称,函数名称

​ + 解析变量的时候,声明被提升了,但是后面的赋值不会进行提升,所以会报出undefined

​ + 函数声明被解析的时候,会提升到最前面,但是不会调用

​ + 在函数中使用变量,切记,切记一定要用var,否则就会被解析成一个全局下的变量

​ + 补充问题:

​ - 预解析,说白了就是一个坑,并且是一个大大坑,后续你用的时候,学会主动的规避这些问题就好了

​ - 但是,建议到课程后去,去刷一些预解析相关的面试题,例如(鹅厂,某东,某想,鹅厂的第二厂)

​ 7- 函数中的arguments对象

​ + 本身是一个伪数组对象

​ - 因为可以循环遍历,但是没有真正数组对象上面的方法

​ + 存在于函数中的一个较为特殊的对象

​ - 本身它还有属于自己的方法

​ + 可以在高程中查阅,或者MDN

​ + 作用:

​ - 用于接收实参传递进来的数据

​ + 在ES6中,有一个叫做箭头函数的函数,这个箭头函数中,就没有这个arguments对象

​ + 记住了,能够说明它的使用场景,以及主要作用,就行

​ 8- 函数中的this出现的场景

​ + 在全局下定义的函数,它的this指向的就是全局最大的对象window

​ + 在对象的方法中,this指向的就是最后的调用者

​ + 在BOM的定时器中,this指向的也还是window

​ + 在DOM中的事件处理函数中,this指向的是事件源本身

​ + 在ES6的箭头函数中,由于箭头函数没有自己的this,它的this来自于外部对象的this

​ + 在nodejs阶段会讲es6,到时候,哼哼~好好听,来自于2019-08-16 :15:50

​ 9- 创建对象的3种方式

​ + 字面量方式创建对象

​ + 工厂模式创建对象

​ + 自定义构造函数创建对象

​ 10- new到底干了几件事

​ + 创建了一个新的空对象

​ + 将this指向这个新的空对象

​ + 把属性和方法给到这个新的对象上

​ + 返回这个对象

+请求(request) 与响应(response)
 +localhost : 本地服务器
 +访问了一个URL地址,确有多个请求 :
           如果一个HTML文件中引用了其他文件,如css/js文件,图片,则会发出多次请求
   +接口 :    接口是服务器上的一段代码,你访问接口就是让服务器去执行这段代码,并返回结果给你.
   +http协议分成两个大的部分:
               -  请求报文:
                     1.请求行: 请求方式 请求路径 协议及版本
                     2.请求头: 自动携带的信息,以键值对格式构成
                     3.请求体: 保存本次附带的数据,如用户名密码等
                -  响应报文:
                     1.请求行: 协议及版本 状态码 说明
                     2.请求头: 服务器返回响应时携带的附加信息,以键值对格式构成
                     3.请求体: 返回主体内容
 +5个常见的状态码:
         200  成功
         302   重定向
         304   读取缓存
         404   请求资源不存在
         500   服务器内部错误

+ajax使用场景:
局部更新 :(不更新页面,数据更新)
+ajax是什么,解决什么:
-ajax : Asynchronous JavaScript And Xml (异步的javascript和xml) 创建交互式网页应用的网页开发技术
- 解决网页局部更新 :(优点)
1.局部更新,提升用户体验
2.分离开发,提高开发效率
+$.ajax参数及含义:
url :请求地址
type : 请求类型, 常见get, post (默认get)
data : 发送请求时附加的参数,
success : 回调函数
+ajax请求接口与普通http请求接口区别 :
ajax : 控制何时发请求, 控制响应结果
普通http : 不能控制何时发请求, 不能处理响应结果
+ajax请求的响应体与其他类型的http请求响应体的区别:
ajax:
1.通过js代码操作响应体
2.响应体内容为数据
普通请求:
1.浏览器直接处理响应体
2.响应体内容为代码
